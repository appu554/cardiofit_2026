# Therapeutic Inertia Detection — Implementation Plan


## Clinical Rationale


**Why this is the single most impactful CDS intervention in cardiometabolic care:**


Therapeutic inertia — the failure to initiate or intensify therapy when clinical targets are not met — is the dominant cause of poor outcomes in T2DM and hypertension management globally. It is not a rare edge case. It is the default behavior of the healthcare system.


The numbers are staggering. Khunti et al. (2018, Diabetes Care) analyzed 105,477 T2DM patients in UK primary care and found that median time from HbA1c first exceeding 7.0% to first treatment intensification was **2.9 years**. For patients exceeding 8.0%, median time to intensification was still **>1.4 years**. Shah et al. (2019, JAMA Internal Medicine) replicated this in a US population of 311,906 patients — median time to intensification after HbA1c >8% was **14.2 months**. During that delay, each patient accumulates approximately 0.4% additional HbA1c exposure per year of inertia, translating directly into microvascular and macrovascular complication risk (UKPDS legacy effect, Holman et al. 2008).


For hypertension, the picture is equally bleak. Okonofua et al. (2006, Hypertension) showed that intensification occurred at only 13.1% of visits where BP was above target. Lebeau et al. (2014) found that among patients with BP consistently >140/90 over 6 months, only 28% had any medication change. The SPRINT trial (2015) demonstrated that intensive SBP control (<120 vs <140) reduced cardiovascular events by 25% and mortality by 27% — but achieving this requires overcoming therapeutic inertia at every visit.


**Why therapeutic inertia happens — and why CDS can address it:**


The causes are well-characterized (Reach et al., 2017, Diabetes & Metabolism; Phillips et al., 2001, Annals of Internal Medicine):


**Physician-side factors:** "Soft reasons" dominate — "let's wait and see," "the patient is doing well otherwise," "we just changed something," "the lab was probably an outlier." These are cognitive biases (status quo bias, omission bias, anchoring to last visit) that CDS can directly counter by presenting objective temporal evidence of persistent uncontrolled status.


**Patient-side factors:** Reluctance to add medications, cost concerns, side effect fears, perceived wellness despite abnormal numbers. The platform can detect these through engagement patterns (Module 9) and medication adherence data.


**System-side factors:** Short visit times, competing clinical priorities, fragmented care. CDS can pre-compute the inertia evidence so the physician has it at the moment of decision, not buried in chart history.


**What the evidence says about CDS interventions for inertia:**


Cimino et al. (2023, Journal of the American Medical Informatics Association) systematically reviewed 42 CDS interventions targeting therapeutic inertia in diabetes. The most effective interventions shared three characteristics: (1) they presented the **duration** of uncontrolled status, not just the current value; (2) they referenced **guideline-specific thresholds** with named recommendations; (3) they provided **similar-patient outcomes** showing what happens when intensification occurs vs. doesn't. This maps directly to our architecture: IOR provides similar-patient outcomes, market configs provide guideline thresholds, and the inertia detector computes duration.


Pantalone et al. (2019, Diabetes Care) demonstrated that an EHR-embedded "time above target" alert reduced median time-to-intensification from 13.2 months to 4.8 months — a 64% reduction. Importantly, the intervention did **not** simply flag "HbA1c is high" (which physicians already know). It flagged "HbA1c has been above target for X months with no treatment change" — the temporal context was the critical differentiator.


**The specific inertia patterns we must detect:**


**Pattern 1 — Glycaemic inertia (HbA1c-based):**
- HbA1c above individualized target (typically 7.0%, relaxed to 8.0% for elderly/frail)
- For ≥2 consecutive HbA1c measurements (typically ≥3 months apart, so ≥6 months of evidence)
- With no MEDICATION_START, DOSE_CHANGE, or MEDICATION_CLASS_ADD event in the IOR system during that period
- This is the canonical inertia pattern studied in Khunti et al. and Shah et al.


**Pattern 2 — Glycaemic inertia (CGM-based, new):**
- When CGM data is available: TIR <50% sustained for ≥14 days (2 sensor cycles)
- Or GRI Zone D/E sustained for ≥14 days
- With no medication change in the period
- This is a V4 innovation — using CGM continuous data to detect inertia faster than the traditional 3-month HbA1c cycle. A patient with TIR 30% for 28 days doesn't need to wait for the next HbA1c to trigger an inertia alert. This alone could reduce detection-to-action time from months to weeks.


**Pattern 3 — Hemodynamic inertia (BP-based):**
- Mean home SBP above target (typically ≥140, or ≥130 for diabetic patients per ISH 2020/RSSDI)
- Sustained for ≥4 weeks with ≥2 readings per week (minimum data quality)
- With no antihypertensive medication change in the period
- Module 7's BP variability metrics provide the data; the inertia detector adds the temporal + medication cross-reference


**Pattern 4 — Dual-domain inertia (the most dangerous):**
- Both glycaemic AND hemodynamic targets unmet simultaneously
- For ≥8 weeks
- With no medication change in either domain
- This is concordant inertia — the risk is multiplicative, not additive. AHA's CKM framework (Ndumele et al., 2023) explicitly identifies concordant uncontrolled cardiometabolic risk as requiring urgent multi-domain intensification.


**Pattern 5 — Post-event inertia:**
- After a clinical event (hospitalization, acute hypo episode, acute hypertensive event)
- With return to outpatient care
- And no medication adjustment within 4 weeks of the event
- Post-event inertia is the highest-risk form — the event itself is evidence that current therapy is inadequate, and delay in adjustment dramatically increases recurrence risk (Feldman et al., 2009)


**Pattern 6 — Renal progression inertia (novel):**
- eGFR decline crossing a CKD stage boundary (e.g., G3a→G3b, meaning eGFR dropped below 45)
- With no medication review in response (no renoprotective agent added, no dose adjustment)
- This connects directly to the renal dose gating system — when eGFR crosses 45, the system should both gate existing medications AND detect whether the clinician responds to the stage transition


**Pattern 7 — Intensification ceiling inertia:**
- Patient on maximum tolerated dose of current medication class
- With targets still unmet
- But no addition of a second/third class agent
- This is distinct from standard inertia — the clinician may have already titrated to maximum, but hasn't taken the next step of adding a complementary class. ADA Standards of Care 2025 Section 9 explicitly recommends early combination therapy when monotherapy at maximum dose doesn't achieve target within 3 months.

**India-Specific Inertia Considerations:**


Deepa et al. (2022, Lancet Diabetes & Endocrinology) analyzed therapeutic inertia in 12,537 Indian T2DM patients across the ICMR-INDIAB study. Findings specific to the Indian context:


- **Median time to intensification after HbA1c >7.5%: 3.4 years** — worse than UK/US data, driven by physician hesitancy to add insulin and patient cost barriers
- **Sulfonylurea-metformin combination remains the dominant second-line** — GLP-1 RA and SGLT2i are underused despite guidelines recommending them, primarily due to cost. The inertia detector must recognize that recommending GLP-1 RA in a tier-2 Indian city may not be inertia — it may be rational cost-constrained decision-making. The system should differentiate between "physician didn't intensify" and "physician couldn't intensify due to formulary/cost constraints"
- **Festival-driven compliance breaks** — Diwali, Holi, Ramadan, wedding seasons cause predictable medication interruptions. An apparent "no medication change" period may include a patient-initiated break that the physician is aware of but hasn't documented. The inertia detector should cross-reference engagement data for adherence dips before flagging.
- **RSSDI 2023 stepped intensification pathway:** Metformin → Metformin + SU/DPP4i → Metformin + SU + DPP4i/SGLT2i → Insulin. The inertia detector should reference this pathway to identify which specific intensification step is being delayed.


**Australia-Specific Inertia Considerations:**


- **RACGP 2024 "Red Book"** recommends HbA1c target review at every GP visit (at least annually). The GP Management Plan (GPMP) and Team Care Arrangement (TCA) under Medicare incentivize regular chronic disease review — but Achat et al. (2022) showed that even with GPMP incentives, therapeutic inertia rates in Australian primary care for T2DM are 45-60%.
- **PBS authority requirements** introduce structural inertia — adding SGLT2i or GLP-1 RA requires a PBS authority prescription, which involves a phone call or online application. Some GPs delay intensification to avoid the administrative burden. The system should flag when PBS authority is the likely barrier and provide the authority item code to reduce friction.
- **Indigenous health:** Remote community patients often see a GP only every 2-4 weeks (fly-in services). The inertia detection window must account for this visit frequency — a 12-week gap between medication changes may be normal for a patient who's only seen a GP 3 times in that period.
- **ADS-ADEA shared care model:** Endocrinologist/diabetes educator may have adjusted therapy but not updated the GP's system. The inertia detector should check for *any* medication change across care providers, not just the primary prescriber.

## Architecture Overview


The therapeutic inertia detection system spans three services and connects to three existing V4 systems:


1. **KB-23 (Decision Cards) — Inertia Detection Engine:** Core engine that cross-references uncontrolled clinical status (from KB-20 V4 state, Module 7 BP metrics, KB-26 CGM analytics) against IOR intervention timestamps. Produces inertia verdicts with evidence chains. Generates inertia-specific decision cards with structured rationale.


2. **KB-20 (Patient Profile) — Intervention Timeline Query:** Exposes endpoint to query the most recent medication change by domain (glycaemic, hemodynamic, renal) and the most recent clinical encounter. The IOR system (Task 8) already stores intervention records — the inertia detector queries against them.


3. **KB-26 (Metabolic Digital Twin) — Target Status Computation:** Computes whether each domain is at target using the most current data available (CGM analytics if available, else FBG/HbA1c for glycaemic; Module 7 BP metrics for hemodynamic; eGFR trajectory for renal). Publishes `TARGET_STATUS_CHANGED` events when a domain transitions from at-target to not-at-target.


**Data flow:**
```
KB-20 V4 State (HbA1c, eGFR, etc.)  ─┐
Module 7 BP Metrics                   ─┤
KB-26 CGM Analytics (TIR, GRI)        ─┤──→  Inertia Detection Engine (KB-23)
KB-26 MHRI Trajectory                 ─┤       ├── Cross-reference with IOR
IOR Intervention Records (KB-20)      ─┘       ├── Compute inertia duration
                                               ├── Classify inertia pattern
Market Config (targets + pathways)             ├── Generate evidence chain
                                               └── Produce inertia cards
```


**The critical insight:** Inertia detection is fundamentally a **temporal join** — joining "domain X has been out of target since timestamp T1" with "the most recent intervention in domain X was at timestamp T0, where T0 < T1." The duration T_now − T1 is the inertia period. The system must compute this join across multiple domains simultaneously and detect concordant inertia (multiple domains simultaneously overdue).

```go
// kb-23-decision-cards/internal/models/therapeutic_inertia.go
package models


import "time"


// InertiaDomain identifies which clinical domain is subject to inertia evaluation.
type InertiaDomain string


const (
    DomainGlycaemic    InertiaDomain = "GLYCAEMIC"
    DomainHemodynamic  InertiaDomain = "HEMODYNAMIC"
    DomainRenal        InertiaDomain = "RENAL"
    DomainLipid        InertiaDomain = "LIPID"
)


// InertiaPattern classifies the type of therapeutic inertia detected.
type InertiaPattern string


const (
    PatternHbA1cInertia         InertiaPattern = "HBA1C_INERTIA"
    PatternCGMInertia           InertiaPattern = "CGM_INERTIA"
    PatternBPInertia            InertiaPattern = "BP_INERTIA"
    PatternDualDomainInertia    InertiaPattern = "DUAL_DOMAIN_INERTIA"
    PatternPostEventInertia     InertiaPattern = "POST_EVENT_INERTIA"
    PatternRenalProgressionInertia InertiaPattern = "RENAL_PROGRESSION_INERTIA"
    PatternIntensificationCeiling  InertiaPattern = "INTENSIFICATION_CEILING"
)


// InertiaVerdict is the output of the inertia detection engine for a single domain.
type InertiaVerdict struct {
    Domain              InertiaDomain  `json:"domain"`
    Pattern             InertiaPattern `json:"pattern"`
    Detected            bool           `json:"detected"`
    InertiaDurationDays int            `json:"inertia_duration_days"`


    // The uncontrolled status evidence
    TargetValue         float64   `json:"target_value"`          // what the target is
    CurrentValue        float64   `json:"current_value"`         // latest measurement
    FirstExceedanceDate time.Time `json:"first_exceedance_date"` // when target was first exceeded
    ConsecutiveReadings int       `json:"consecutive_readings"`  // how many readings above target
    DataSource          string    `json:"data_source"`           // HBA1C, CGM_TIR, HOME_BP, LAB_EGFR


    // The intervention gap evidence
    LastInterventionDate *time.Time `json:"last_intervention_date,omitempty"`
    LastInterventionType string     `json:"last_intervention_type,omitempty"` // MEDICATION_START, DOSE_CHANGE, CLASS_ADD
    DaysSinceIntervention int       `json:"days_since_intervention"`


    // Context
    CurrentMedications   []string `json:"current_medications"`
    AtMaxDose            bool     `json:"at_max_dose,omitempty"`     // for ceiling inertia
    NextStepInPathway    string   `json:"next_step_in_pathway"`     // guideline-recommended next step
    CostBarrierLikely    bool     `json:"cost_barrier_likely"`      // India: expensive next-step
    PBSAuthorityRequired bool     `json:"pbs_authority_required"`   // Australia: PBS barrier


    // Severity
    Severity            string `json:"severity"`  // MILD (6-12 wk), MODERATE (12-26 wk), SEVERE (26-52 wk), CRITICAL (>52 wk)
    RiskAccumulation    string `json:"risk_accumulation"` // estimated additional risk from delay


    // Guideline reference
    GuidelineReference  string `json:"guideline_reference"`
}


// PatientInertiaReport is the full inertia assessment for a patient across all domains.
type PatientInertiaReport struct {
    PatientID            string           `json:"patient_id"`
    EvaluatedAt          time.Time        `json:"evaluated_at"`
    Verdicts             []InertiaVerdict `json:"verdicts"`
    HasAnyInertia        bool             `json:"has_any_inertia"`
    HasDualDomainInertia bool             `json:"has_dual_domain_inertia"`
    MostSevere           *InertiaVerdict  `json:"most_severe,omitempty"`
    OverallUrgency       string           `json:"overall_urgency"`
    InertiaScore         float64          `json:"inertia_score"` // 0-100 composite
}


// DomainTargetStatus tracks whether a domain is at target, and for how long it hasn't been.
type DomainTargetStatus struct {
    Domain              InertiaDomain `json:"domain"`
    AtTarget            bool          `json:"at_target"`
    CurrentValue        float64       `json:"current_value"`
    TargetValue         float64       `json:"target_value"`
    FirstUncontrolledAt *time.Time    `json:"first_uncontrolled_at,omitempty"`
    DaysUncontrolled    int           `json:"days_uncontrolled"`
    ConsecutiveReadings int           `json:"consecutive_readings"`
    DataSource          string        `json:"data_source"`
    Confidence          string        `json:"confidence"` // HIGH, MODERATE, LOW
}


// InterventionTimeline represents the most recent interventions in each domain.
type InterventionTimeline struct {
    PatientID  string                          `json:"patient_id"`
    ByDomain   map[InertiaDomain]LatestAction  `json:"by_domain"`
    AnyChangeInLast12Weeks bool               `json:"any_change_in_last_12_weeks"`
    TotalActiveInterventions int              `json:"total_active_interventions"`
}


// LatestAction is the most recent intervention in a specific domain.
type LatestAction struct {
    InterventionID   string    `json:"intervention_id"`
    InterventionType string    `json:"intervention_type"`
    DrugClass        string    `json:"drug_class,omitempty"`
    DrugName         string    `json:"drug_name,omitempty"`
    DoseMg           float64   `json:"dose_mg,omitempty"`
    ActionDate       time.Time `json:"action_date"`
    DaysSince        int       `json:"days_since"`
}


// IntensificationPathway represents the guideline-recommended treatment escalation for a domain.
type IntensificationStep struct {
    StepNumber  int      `yaml:"step" json:"step"`
    DrugClasses []string `yaml:"drug_classes" json:"drug_classes"`
    Description string   `yaml:"description" json:"description"`
    MaxDoseMg   map[string]float64 `yaml:"max_dose_mg" json:"max_dose_mg,omitempty"`
}
```

# Sources: ADA SOC 2025, ESC/EASD 2023, KDIGO 2024, Khunti et al. 2018


# Inertia evaluation windows — minimum durations before flagging

  renal_inertia:

    max_response_weeks: 4              # expect medication review within 4 weeks of event

      - "HYPOGLYCAEMIA_SEVERE"
      - "HYPERTENSIVE_CRISIS"
      - "HOSPITALIZATION_CV"
      - "HOSPITALIZATION_RENAL"
      - "DKA"


# Severity classification by inertia duration
severity:
  mild_weeks: 12         # 12-26 weeks
  moderate_weeks: 26     # 26-52 weeks

    per_year_egfr_decline_unaddressed: "Unaddressed CKD progression: +15% ESRD risk per year"

  - "PALLIATIVE_CARE"                     # comfort-focused care
  - "LIFE_EXPECTANCY_LESS_THAN_1_YEAR"

  - "SPECIALIST_REVIEW_PENDING"           # referral made, awaiting specialist

// kb-20-patient-profile/internal/services/intervention_timeline.go
package services


import (
    "time"


    "gorm.io/gorm"
    "kb-patient-profile/internal/models"
)


// Drug class → primary clinical domain mapping
var drugClassDomainMap = map[string]string{
    "METFORMIN":       "GLYCAEMIC",
    "SULFONYLUREA":    "GLYCAEMIC",
    "DPP4i":           "GLYCAEMIC",
    "SGLT2i":          "GLYCAEMIC",
    "GLP1_RA":         "GLYCAEMIC",
    "INSULIN":         "GLYCAEMIC",
    "BASAL_INSULIN":   "GLYCAEMIC",
    "PIOGLITAZONE":    "GLYCAEMIC",
    "EXENATIDE":       "GLYCAEMIC",
    "ACEi":            "HEMODYNAMIC",
    "ARB":             "HEMODYNAMIC",
    "CCB":             "HEMODYNAMIC",
    "AMLODIPINE":      "HEMODYNAMIC",
    "THIAZIDE":        "HEMODYNAMIC",
    "LOOP_DIURETIC":   "HEMODYNAMIC",
    "BETA_BLOCKER":    "HEMODYNAMIC",
    "MRA":             "HEMODYNAMIC",
    "SPIRONOLACTONE":  "HEMODYNAMIC",
    "STATIN":          "LIPID",
    "EZETIMIBE":       "LIPID",
    "FINERENONE":      "RENAL",
}


func MapDrugClassToDomain(drugClass string) string {
    if d, ok := drugClassDomainMap[drugClass]; ok {
        return d
    }
    return "OTHER"
}


// InterventionTimelineResult is the timeline response.
type InterventionTimelineResult struct {
    PatientID                string                           `json:"patient_id"`
    ByDomain                 map[string]LatestDomainAction    `json:"by_domain"`
    AnyChangeInLast12Weeks   bool                             `json:"any_change_in_last_12_weeks"`
    TotalActiveInterventions int                              `json:"total_active_interventions"`
}


type LatestDomainAction struct {
    InterventionID   string    `json:"intervention_id"`
    InterventionType string    `json:"intervention_type"`
    DrugClass        string    `json:"drug_class"`
    DrugName         string    `json:"drug_name,omitempty"`
    DoseMg           float64   `json:"dose_mg,omitempty"`
    ActionDate       time.Time `json:"action_date"`
    DaysSince        int       `json:"days_since"`
}


func (s *IORStore) GetInterventionTimeline(patientID string) (*InterventionTimelineResult, error) {
    var interventions []models.InterventionRecord
    if err := s.db.Where("patient_id = ? AND status IN ('ACTIVE', 'COMPLETED')", patientID).
        Order("start_date DESC").Find(&interventions).Error; err != nil {
        return nil, err
    }


    now := time.Now()
    twelveWeeksAgo := now.Add(-12 * 7 * 24 * time.Hour)


    result := &InterventionTimelineResult{
        PatientID: patientID,
        ByDomain:  make(map[string]LatestDomainAction),
    }


    anyRecent := false
    activeCount := 0


    for _, iv := range interventions {
        if iv.Status == "ACTIVE" {
            activeCount++
        }


        drugClass := ""
        if iv.DrugClass != nil {
            drugClass = *iv.DrugClass
        }
        domain := MapDrugClassToDomain(drugClass)
        if domain == "OTHER" {
            continue
        }


        // Only keep the most recent per domain (interventions are sorted DESC)
        if _, exists := result.ByDomain[domain]; !exists {
            drugName := ""
            if iv.DrugName != nil {
                drugName = *iv.DrugName
            }
            doseMg := 0.0
            if iv.DoseChangeMg != nil {
                doseMg = *iv.DoseChangeMg
            }


            result.ByDomain[domain] = LatestDomainAction{
                InterventionID:   iv.ID.String(),
                InterventionType: iv.InterventionType,
                DrugClass:        drugClass,
                DrugName:         drugName,
                DoseMg:           doseMg,
                ActionDate:       iv.StartDate,
                DaysSince:        int(now.Sub(iv.StartDate).Hours() / 24),
            }
        }


        if iv.StartDate.After(twelveWeeksAgo) {
            anyRecent = true
        }
    }


    result.AnyChangeInLast12Weeks = anyRecent
    result.TotalActiveInterventions = activeCount


    return result, nil
}

```go
// kb-26-metabolic-digital-twin/internal/services/target_status.go
package services


import (
    "time"
)


// TargetStatusInput for glycaemic domain evaluation.
type TargetStatusInput struct {
    PatientID       string
    HbA1c           *float64
    HbA1cDate       *time.Time
    PrevHbA1c       *float64
    PrevHbA1cDate   *time.Time
    HbA1cTarget     float64
    CGMAvailable    bool
    CGMTIR          *float64
    CGMGRIZone      string
    CGMReportDate   *time.Time
    CGMSufficientData bool
    TIRTarget       float64
}


// BPTargetStatusInput for hemodynamic domain evaluation.
type BPTargetStatusInput struct {
    PatientID        string
    MeanSBP7d        *float64
    MeanDBP7d        *float64
    BPDataDate       *time.Time
    SBPTarget        float64
    DBPTarget        float64
    WeeksAboveTarget int
    ReadingsInPeriod int
}


// RenalTargetStatusInput for renal domain evaluation.
type RenalTargetStatusInput struct {
    PatientID         string
    CurrentEGFR       float64
    PreviousEGFR      float64
    CurrentCKDStage   string
    PreviousCKDStage  string
    StageTransition   bool
    TransitionDate    *time.Time
}


// DomainTargetStatusResult is the output.
type DomainTargetStatusResult struct {
    Domain              string     `json:"domain"`
    AtTarget            bool       `json:"at_target"`
    CurrentValue        float64    `json:"current_value"`
    TargetValue         float64    `json:"target_value"`
    FirstUncontrolledAt *time.Time `json:"first_uncontrolled_at,omitempty"`
    DaysUncontrolled    int        `json:"days_uncontrolled"`
    ConsecutiveReadings int        `json:"consecutive_readings"`
    DataSource          string     `json:"data_source"`
    Confidence          string     `json:"confidence"`
}


// ComputeGlycaemicTargetStatus evaluates whether the glycaemic domain is at target.
// Prefers CGM data when available and sufficient; falls back to HbA1c.
func ComputeGlycaemicTargetStatus(input TargetStatusInput) DomainTargetStatusResult {
    result := DomainTargetStatusResult{Domain: "GLYCAEMIC"}


    // Prefer CGM if available with sufficient data
    if input.CGMAvailable && input.CGMSufficientData && input.CGMTIR != nil {
        result.DataSource = "CGM_TIR"
        result.CurrentValue = *input.CGMTIR
        result.TargetValue = input.TIRTarget
        result.AtTarget = *input.CGMTIR >= input.TIRTarget
        result.Confidence = "HIGH"


        if !result.AtTarget && input.CGMReportDate != nil {
            result.FirstUncontrolledAt = input.CGMReportDate
            result.DaysUncontrolled = int(time.Now().Sub(*input.CGMReportDate).Hours() / 24)
            result.ConsecutiveReadings = 1 // CGM is continuous — treat as one sustained reading
        }
        return result
    }


    // Fall back to HbA1c
    result.DataSource = "HBA1C"
    result.Confidence = "MODERATE"


    if input.HbA1c == nil {
        result.Confidence = "LOW"
        result.AtTarget = true // assume target if no data — don't flag inertia without evidence
        return result
    }


    result.CurrentValue = *input.HbA1c
    result.TargetValue = input.HbA1cTarget
    result.AtTarget = *input.HbA1c <= input.HbA1cTarget


    if !result.AtTarget {
        consecutive := 1
        firstExceedance := input.HbA1cDate


        // Check if previous HbA1c was also above target
        if input.PrevHbA1c != nil && *input.PrevHbA1c > input.HbA1cTarget {
            consecutive = 2
            firstExceedance = input.PrevHbA1cDate
        }


        result.ConsecutiveReadings = consecutive
        result.FirstUncontrolledAt = firstExceedance
        if firstExceedance != nil {
            result.DaysUncontrolled = int(time.Now().Sub(*firstExceedance).Hours() / 24)
        }
    }


    return result
}


// ComputeHemodynamicTargetStatus evaluates whether BP is at target.
func ComputeHemodynamicTargetStatus(input BPTargetStatusInput) DomainTargetStatusResult {
    result := DomainTargetStatusResult{
        Domain:     "HEMODYNAMIC",
        DataSource: "HOME_BP",
        Confidence: "HIGH",
    }


    if input.MeanSBP7d == nil {
        result.AtTarget = true
        result.Confidence = "LOW"
        return result
    }


    result.CurrentValue = *input.MeanSBP7d
    result.TargetValue = input.SBPTarget
    result.AtTarget = *input.MeanSBP7d <= input.SBPTarget


    if !result.AtTarget {
        result.DaysUncontrolled = input.WeeksAboveTarget * 7
        result.ConsecutiveReadings = input.ReadingsInPeriod


        if input.BPDataDate != nil {
            weeksAgo := time.Now().Add(-time.Duration(input.WeeksAboveTarget) * 7 * 24 * time.Hour)
            result.FirstUncontrolledAt = &weeksAgo
        }
    }


    // DBP check — flag if DBP also above target
    if input.MeanDBP7d != nil && *input.MeanDBP7d > input.DBPTarget && !result.AtTarget {
        // Already flagged via SBP — DBP adds severity context but doesn't change AtTarget
    }


    return result
}


// ComputeRenalTargetStatus evaluates whether renal function is stable or declining.
func ComputeRenalTargetStatus(input RenalTargetStatusInput) DomainTargetStatusResult {
    result := DomainTargetStatusResult{
        Domain:       "RENAL",
        DataSource:   "LAB_EGFR",
        CurrentValue: input.CurrentEGFR,
        TargetValue:  input.PreviousEGFR, // "target" is at least maintaining previous level
        Confidence:   "MODERATE",
    }


    if input.StageTransition {
        result.AtTarget = false
        result.ConsecutiveReadings = 2 // at minimum, two readings showing decline
        if input.TransitionDate != nil {
            result.FirstUncontrolledAt = input.TransitionDate
            result.DaysUncontrolled = int(time.Now().Sub(*input.TransitionDate).Hours() / 24)
        }
    } else {
        result.AtTarget = true // no stage transition — stable
    }


    return result
}

// kb-23-decision-cards/internal/services/inertia_detector.go
package services


import (
    "fmt"
    "time"


    "kb-decision-cards/internal/models"
)


type InertiaConfig struct {
    TitrationGracePeriodWeeks int
    HbA1cMinDurationWeeks     int
    CGMMinDays                int
    BPMinWeeks                int
    MildWeeks                 int
    ModerateWeeks             int
    SevereWeeks               int
    CriticalWeeks             int
}


type InertiaDetector struct {
    config *InertiaConfig
}


func NewInertiaDetector(config *InertiaConfig) *InertiaDetector {
    return &InertiaDetector{config: config}
}


// EvaluateDomain evaluates therapeutic inertia for a single domain.
func (d *InertiaDetector) EvaluateDomain(
    target models.DomainTargetStatus,
    timeline *models.InterventionTimeline,
) models.InertiaVerdict {
    return d.EvaluateDomainWithMaxDoseInfo(target, timeline, nil)
}


func (d *InertiaDetector) EvaluateDomainWithMaxDoseInfo(
    target models.DomainTargetStatus,
    timeline *models.InterventionTimeline,
    maxDoses map[string]float64,
) models.InertiaVerdict {


    verdict := models.InertiaVerdict{
        Domain:       target.Domain,
        TargetValue:  target.TargetValue,
        CurrentValue: target.CurrentValue,
        DataSource:   target.DataSource,
    }


    // If at target, no inertia possible
    if target.AtTarget {
        verdict.Detected = false
        return verdict
    }


    // Get latest intervention for this domain
    domainAction, hasAction := timeline.ByDomain[target.Domain]


    // Check titration grace period
    if hasAction && domainAction.DaysSince < d.config.TitrationGracePeriodWeeks*7 {
        verdict.Detected = false
        return verdict
    }


    // Determine minimum inertia duration threshold based on data source
    var minDurationDays int
    switch target.DataSource {
    case "CGM_TIR":
        minDurationDays = d.config.CGMMinDays
        verdict.Pattern = models.PatternCGMInertia
    case "HOME_BP":
        minDurationDays = d.config.BPMinWeeks * 7
        verdict.Pattern = models.PatternBPInertia
    case "HBA1C":
        minDurationDays = d.config.HbA1cMinDurationWeeks * 7
        verdict.Pattern = models.PatternHbA1cInertia
    case "LAB_EGFR":
        minDurationDays = 28 // 4 weeks for renal stage transition
        verdict.Pattern = models.PatternRenalProgressionInertia
    default:
        minDurationDays = d.config.HbA1cMinDurationWeeks * 7
        verdict.Pattern = models.PatternHbA1cInertia
    }


    // Check if uncontrolled duration exceeds minimum
    if target.DaysUncontrolled < minDurationDays {
        verdict.Detected = false
        return verdict
    }


    // Check for intensification ceiling
    if hasAction && maxDoses != nil {
        maxDose, hasMax := maxDoses[domainAction.DrugClass]
        if hasMax && domainAction.DoseMg >= maxDose {
            verdict.Pattern = models.PatternIntensificationCeiling
            verdict.AtMaxDose = true
        }
    }


    // Inertia detected
    verdict.Detected = true
    verdict.InertiaDurationDays = target.DaysUncontrolled
    verdict.ConsecutiveReadings = target.ConsecutiveReadings
    verdict.FirstExceedanceDate = safeDeref(target.FirstUncontrolledAt)


    if hasAction {
        verdict.LastInterventionDate = &domainAction.ActionDate
        verdict.LastInterventionType = domainAction.InterventionType
        verdict.DaysSinceIntervention = domainAction.DaysSince
        verdict.CurrentMedications = []string{domainAction.DrugClass}
    }


    // Classify severity
    weeks := target.DaysUncontrolled / 7
    switch {
    case weeks >= d.config.CriticalWeeks:
        verdict.Severity = "CRITICAL"
    case weeks >= d.config.SevereWeeks:
        verdict.Severity = "SEVERE"
    case weeks >= d.config.ModerateWeeks:
        verdict.Severity = "MODERATE"
    default:
        verdict.Severity = "MILD"
    }


    // Determine next step in pathway
    verdict.NextStepInPathway = d.recommendNextStep(target.Domain, domainAction.DrugClass, verdict.AtMaxDose)


    return verdict
}


// EvaluatePatient runs inertia detection across all domains and detects dual-domain inertia.
func (d *InertiaDetector) EvaluatePatient(
    patientID string,
    targetStatuses []models.DomainTargetStatus,
    timeline *models.InterventionTimeline,
) models.PatientInertiaReport {


    report := models.PatientInertiaReport{
        PatientID:   patientID,
        EvaluatedAt: time.Now(),
    }


    var inertiaCount int
    var glycaemicInertia, hemodynamicInertia bool
    var mostSevereScore int


    for _, ts := range targetStatuses {
        verdict := d.EvaluateDomain(ts, timeline)
        report.Verdicts = append(report.Verdicts, verdict)


        if verdict.Detected {
            inertiaCount++
            report.HasAnyInertia = true


            severityScore := severityToScore(verdict.Severity)
            if severityScore > mostSevereScore {
                mostSevereScore = severityScore
                v := verdict
                report.MostSevere = &v
            }


            if verdict.Domain == models.DomainGlycaemic {
                glycaemicInertia = true
            }
            if verdict.Domain == models.DomainHemodynamic {
                hemodynamicInertia = true
            }
        }
    }


    // Dual-domain inertia detection
    if glycaemicInertia && hemodynamicInertia {
        report.HasDualDomainInertia = true


        // Create dual-domain verdict
        dualVerdict := models.InertiaVerdict{
            Domain:   "DUAL",
            Pattern:  models.PatternDualDomainInertia,
            Detected: true,
            Severity: "CRITICAL",
        }
        report.Verdicts = append(report.Verdicts, dualVerdict)
    }


    // Overall urgency
    if report.HasDualDomainInertia {
        report.OverallUrgency = "IMMEDIATE"
    } else if mostSevereScore >= 3 {
        report.OverallUrgency = "IMMEDIATE"
    } else if mostSevereScore >= 2 {
        report.OverallUrgency = "URGENT"
    } else if report.HasAnyInertia {
        report.OverallUrgency = "ROUTINE"
    } else {
        report.OverallUrgency = "NONE"
    }


    // Composite inertia score (0-100)
    report.InertiaScore = d.computeInertiaScore(report)


    return report
}


func (d *InertiaDetector) recommendNextStep(domain models.InertiaDomain, currentDrug string, atMax bool) string {
    switch domain {
    case models.DomainGlycaemic:
        if atMax && currentDrug == "METFORMIN" {
            return "Add SGLT2i (preferred if CKD/CVD) or DPP4i/SU — ADA SOC 2025 Step 2"
        }
        if currentDrug == "METFORMIN" {
            return "Titrate metformin to max 2000mg/day, or add SGLT2i/GLP-1 RA"
        }
        if currentDrug == "SULFONYLUREA" {
            return "Add SGLT2i or DPP4i; consider insulin if triple oral fails"
        }
        return "Review current regimen and consider intensification per guidelines"


    case models.DomainHemodynamic:
        if currentDrug == "AMLODIPINE" || currentDrug == "CCB" {
            return "Add ACEi/ARB (preferred first-line for DM + HTN — renoprotective)"
        }
        if currentDrug == "ACEi" || currentDrug == "ARB" {
            return "Add CCB (amlodipine) — ISH 2020 Step 2"
        }
        return "Review antihypertensive regimen — ISH 2020 stepped pathway"


    case models.DomainRenal:
        return "Add renoprotective agent: SGLT2i (if eGFR ≥20) or finerenone (if eligible)"
    }


    return "Consult guidelines for next intensification step"
}


func severityToScore(severity string) int {
    switch severity {
    case "CRITICAL":
        return 4
    case "SEVERE":
        return 3
    case "MODERATE":
        return 2
    case "MILD":
        return 1
    default:
        return 0
    }
}


func (d *InertiaDetector) computeInertiaScore(report models.PatientInertiaReport) float64 {
    if !report.HasAnyInertia {
        return 0
    }


    score := 0.0
    for _, v := range report.Verdicts {
        if !v.Detected {
            continue
        }
        domainWeight := 1.0
        if v.Domain == models.DomainGlycaemic || v.Domain == models.DomainHemodynamic {
            domainWeight = 1.5 // primary domains weighted higher
        }
        score += float64(severityToScore(v.Severity)) * 20 * domainWeight
    }


    if report.HasDualDomainInertia {
        score *= 1.5 // concordant inertia amplification
    }


    if score > 100 {
        score = 100
    }
    return score
}


func safeDeref(t *time.Time) time.Time {
    if t == nil {
        return time.Time{}
    }
    return *t
}

// kb-23-decision-cards/internal/services/inertia_evidence_builder.go
package services


import (
    "fmt"


    "kb-decision-cards/internal/models"
)


// EvidenceChain is the structured rationale for an inertia card.
type EvidenceChain struct {
    Summary            string           `json:"summary"`
    TimelineEvents     []TimelineEvent  `json:"timeline_events"`
    RiskStatement      string           `json:"risk_statement"`
    GuidelineReference string           `json:"guideline_reference"`
    RecommendedAction  string           `json:"recommended_action"`
    SimilarPatientNote string           `json:"similar_patient_note,omitempty"`
}


type TimelineEvent struct {
    Label     string `json:"label"`
    DaysAgo   int    `json:"days_ago"`
    Detail    string `json:"detail"`
}


func BuildEvidenceChain(verdict models.InertiaVerdict) EvidenceChain {
    chain := EvidenceChain{}


    switch verdict.Pattern {
    case models.PatternHbA1cInertia:
        chain.Summary = fmt.Sprintf(
            "HbA1c has been above target (%.1f%% vs target ≤%.1f%%) for %d days "+
                "(%d consecutive readings above target). %d days since last medication change.",
            verdict.CurrentValue, verdict.TargetValue,
            verdict.InertiaDurationDays, verdict.ConsecutiveReadings,
            verdict.DaysSinceIntervention)
        chain.GuidelineReference = "ADA Standards of Care 2025, Section 9: If HbA1c target not met after 3 months on current therapy, intensify treatment."
        chain.RiskStatement = fmt.Sprintf(
            "Each year at HbA1c >7%%: +37%% microvascular risk, +14%% MI risk (UKPDS). "+
                "Current delay of %d days accumulates preventable risk.", verdict.InertiaDurationDays)


    case models.PatternCGMInertia:
        chain.Summary = fmt.Sprintf(
            "CGM Time-in-Range %.0f%% (target ≥%.0f%%) sustained for %d days with no medication adjustment. "+
                "CGM provides earlier detection of glycaemic deterioration than quarterly HbA1c.",
            verdict.CurrentValue, verdict.TargetValue, verdict.InertiaDurationDays)
        chain.GuidelineReference = "International Consensus on CGM (Battelino 2023): TIR <50% for >14 days warrants therapy review."
        chain.RiskStatement = "Sustained low TIR indicates persistent hyperglycaemia with accumulating microvascular risk."


    case models.PatternBPInertia:
        chain.Summary = fmt.Sprintf(
            "Mean home SBP %.0f mmHg (target ≤%.0f) for %d days "+
                "with no antihypertensive adjustment. %d days since last medication change.",
            verdict.CurrentValue, verdict.TargetValue,
            verdict.InertiaDurationDays, verdict.DaysSinceIntervention)
        chain.GuidelineReference = "ISH 2020 / ESC 2024: Reassess antihypertensive therapy at 4-week intervals until target achieved."
        chain.RiskStatement = fmt.Sprintf(
            "Each 10 mmHg above target: +30%% stroke risk, +20%% CHD risk (PSC). "+
                "Current excess of %.0f mmHg over %d days is preventable exposure.",
            verdict.CurrentValue-verdict.TargetValue, verdict.InertiaDurationDays)


    case models.PatternDualDomainInertia:
        chain.Summary = "Concordant therapeutic inertia detected: both glycaemic and hemodynamic " +
            "domains are above target with no medication change in either domain."
        chain.RiskStatement = "Concordant uncontrolled cardiometabolic risk is multiplicative — " +
            "combined risk exceeds the sum of individual domain risks (AHA CKM Framework, Ndumele 2023)."
        chain.GuidelineReference = "AHA CKM Syndrome Advisory 2023: Concordant multi-domain deterioration requires urgent multi-domain intensification."


    case models.PatternIntensificationCeiling:
        chain.Summary = fmt.Sprintf(
            "Patient at maximum dose of %s but target not met (%.1f vs ≤%.1f). "+
                "No additional medication class added in %d days.",
            joinMeds(verdict.CurrentMedications), verdict.CurrentValue, verdict.TargetValue,
            verdict.DaysSinceIntervention)
        chain.GuidelineReference = "ADA SOC 2025 Section 9: If target not achieved on maximum dose monotherapy within 3 months, add complementary agent."


    case models.PatternRenalProgressionInertia:
        chain.Summary = "CKD stage progression detected without corresponding medication review. " +
            "Renoprotective therapy should be initiated or intensified."
        chain.GuidelineReference = "KDIGO 2024: CKD stage transitions require immediate medication review including RAS blockade and SGLT2i initiation."
    }


    chain.RecommendedAction = verdict.NextStepInPathway


    // Build timeline events
    if verdict.DaysSinceIntervention > 0 {
        chain.TimelineEvents = append(chain.TimelineEvents, TimelineEvent{
            Label:   "Last medication change",
            DaysAgo: verdict.DaysSinceIntervention,
            Detail:  fmt.Sprintf("%s: %s", verdict.LastInterventionType, joinMeds(verdict.CurrentMedications)),
        })
    }
    if verdict.InertiaDurationDays > 0 {
        chain.TimelineEvents = append(chain.TimelineEvents, TimelineEvent{
            Label:   "First exceeded target",
            DaysAgo: verdict.InertiaDurationDays,
            Detail:  fmt.Sprintf("%.1f (target ≤%.1f)", verdict.CurrentValue, verdict.TargetValue),
        })
    }


    return chain
}


func joinMeds(meds []string) string {
    if len(meds) == 0 {
        return "unknown"
    }
    result := meds[0]
    for _, m := range meds[1:] {
        result += " + " + m
    }
    return result
}

// kb-23-decision-cards/internal/services/inertia_card_generator.go
package services


import (
    "fmt"


    "kb-decision-cards/internal/models"
)


type InertiaCard struct {
    CardType      string        `json:"card_type"`
    Urgency       string        `json:"urgency"`
    Title         string        `json:"title"`
    Rationale     string        `json:"rationale"`
    EvidenceChain EvidenceChain `json:"evidence_chain"`
    Actions       []string      `json:"actions"`
    Domain        string        `json:"domain"`
    InertiaScore  float64       `json:"inertia_score"`
}


func GenerateInertiaCards(report models.PatientInertiaReport) []InertiaCard {
    if !report.HasAnyInertia {
        return nil
    }


    var cards []InertiaCard


    for _, verdict := range report.Verdicts {
        if !verdict.Detected {
            continue
        }


        evidence := BuildEvidenceChain(verdict)


        if verdict.Pattern == models.PatternDualDomainInertia {
            cards = append(cards, InertiaCard{
                CardType: "DUAL_DOMAIN_INERTIA",
                Urgency:  "IMMEDIATE",
                Title:    "Concordant Therapeutic Inertia — Glycaemic + Hemodynamic",
                Rationale: evidence.Summary,
                EvidenceChain: evidence,
                Actions: []string{
                    "Comprehensive medication review across both domains",
                    "Consider dual-benefit agents (SGLT2i for glucose + BP + renal protection)",
                    "Schedule urgent follow-up within 2 weeks",
                },
                Domain:       "DUAL",
                InertiaScore: report.InertiaScore,
            })
            continue
        }


        urgency := verdictToUrgency(verdict)


        card := InertiaCard{
            CardType:      "THERAPEUTIC_INERTIA",
            Urgency:       urgency,
            Title:         buildInertiaTitle(verdict),
            Rationale:     evidence.Summary,
            EvidenceChain: evidence,
            Actions:       buildInertiaActions(verdict),
            Domain:        string(verdict.Domain),
            InertiaScore:  report.InertiaScore,
        }


        cards = append(cards, card)
    }


    return cards
}


func verdictToUrgency(v models.InertiaVerdict) string {
    switch v.Severity {
    case "CRITICAL":
        return "IMMEDIATE"
    case "SEVERE":
        return "URGENT"
    case "MODERATE":
        return "URGENT"
    default:
        return "ROUTINE"
    }
}


func buildInertiaTitle(v models.InertiaVerdict) string {
    domainLabel := map[models.InertiaDomain]string{
        models.DomainGlycaemic:   "Glycaemic",
        models.DomainHemodynamic: "Blood Pressure",
        models.DomainRenal:       "Renal",
    }
    label := domainLabel[v.Domain]


    switch v.Pattern {
    case models.PatternIntensificationCeiling:
        return fmt.Sprintf("%s Target Not Met — At Maximum Dose, Add New Class", label)
    case models.PatternCGMInertia:
        return fmt.Sprintf("CGM Shows Sustained Low Time-in-Range — %s Intensification Needed", label)
    default:
        return fmt.Sprintf("%s Therapeutic Inertia — %d Days Above Target Without Change", label, v.InertiaDurationDays)
    }
}


func buildInertiaActions(v models.InertiaVerdict) []string {
    actions := []string{v.NextStepInPathway}


    if v.Severity == "SEVERE" || v.Severity == "CRITICAL" {
        actions = append(actions, "Consider specialist referral if target persistently unmet")
    }


    if v.AtMaxDose {
        actions = append(actions, "Current medication at maximum dose — add complementary class rather than further dose increase")
    }


    if v.CostBarrierLikely {
        actions = append(actions, "Cost barrier likely — consider affordable alternatives from formulary")
    }


    if v.PBSAuthorityRequired {
        actions = append(actions, "PBS authority required for next-step medication — authority can be obtained online")
    }


    return actions
}
```

# KB-5: Drug–Drug Interactions — Final Proposal (v1.0)


## 1) Purpose & Scope (Reviewed ✅)


* **Purpose**: Authoritative, low-latency DDI service for **CAE in Safety Gateway** (Validate phase) and **CandidateBuilder** (early hard-stop screening).&#x20;
* **In scope**: pairwise DDIs, **class↔drug** and **class↔class** rules, **pharmacogenomics (PGx)**, **food/alcohol/herbal** modifiers, **mechanism, clinical effects, management** with severity + confidence.
* **Out of scope**: dosing (KB-1), absolute non-interaction-based contraindications (KB-4), guideline selection logic (KB-3).&#x20;


### What’s new


* **Context qualifiers**: route, formulation, **dose thresholds**, chronic vs. acute use.
* **Patient-modifiers**: age bands, hepatic/renal stage (for mechanism relevance), **PGx phenotypes**.
* **Evidence & confidence**: level/grade + **provenance** and **recency** fields (ties into Evidence Envelope).

### 2.1 Core Tables (DDL excerpts)


```sql
-- Canonical concepts are RxNorm (ingredient/clinicalDrug) and ATC for classes (via KB-7).


create type ddi_severity as enum ('contraindicated','major','moderate','minor','unknown');
create type evidence_level as enum ('A','B','C','D','ExpertOpinion','Unknown');
create type mechanism_type as enum ('PK','PD','PK_PD','Unknown');
create type context_use as enum ('acute','chronic','unspecified');


create table ddi_interactions (
  id bigserial primary key,
  dataset_version text not null,                  -- ties to Evidence Envelope
  drug1_code text not null,                       -- RxCUI (ingredient or clinical drug)
  drug2_code text not null,                       -- normalized & ordered (min, max)
  severity ddi_severity not null,
  mechanism mechanism_type not null,
  mechanism_note text,
  clinical_effects text,
  management_strategy text not null,
  evidence evidence_level not null,
  confidence numeric(3,2) default 0.80,           -- 0-1 confidence from vendor harmonizer
  route_restriction text[],                       -- e.g., ['oral','iv']
  min_dose1_mg numeric,                           -- dose thresholds (optional)
  min_dose2_mg numeric,
  context context_use default 'unspecified',
  pgx_required boolean default false,
  pgx_markers jsonb,                              -- e.g., {"CYP2D6":"ultrarapid"}
  food_alcohol_herbal jsonb,                      -- {"grapefruit":"avoid","ethanol":"caution"}
  contraindication_reason text,
  source_vendor_ids text[],                       -- lineage/provenance
  created_at timestamptz default now(),
  unique (dataset_version, drug1_code, drug2_code)
);


create table ddi_class_rules (
  id bigserial primary key,
  dataset_version text not null,
  object_type text not null check (object_type in ('drug','class')),
  object_code text not null,                      -- RxCUI or ATC
  subject_type text not null check (subject_type in ('drug','class')),
  subject_code text not null,
  severity ddi_severity not null,
  mechanism mechanism_type not null,
  clinical_effects text,
  management_strategy text not null,
  qualifiers jsonb,                               -- e.g., {"route":"oral","dose1_mg":">=40"}
  evidence evidence_level not null,
  confidence numeric(3,2) default 0.75,
  source_vendor_ids text[],
  unique (dataset_version, object_type, object_code, subject_type, subject_code)
);


create table ddi_pharmacogenomic_rules (
  id bigserial primary key,
  dataset_version text not null,
  drug_code text not null,
  gene text not null,
  phenotype text not null,                        -- e.g., "CYP2D6_ultrarapid"
  interaction_with text,                          -- optional second drug/class
  severity ddi_severity not null,
  clinical_effects text,
  management_strategy text not null,
  evidence evidence_level not null,
  source_vendor_ids text[],
  unique (dataset_version, drug_code, gene, phenotype, coalesce(interaction_with,'NA'))
);


create table ddi_modifiers (
  id bigserial primary key,
  dataset_version text not null,
  modifier_type text not null check (modifier_type in ('food','alcohol','herbal','disease')),
  modifier_code text,                             -- optional controlled term (SNOMED, etc.)
  drug_code text not null,
  effect text not null,                           -- e.g., "↑AUC", "risk_of_bleeding↑"
  management_strategy text not null,
  severity ddi_severity not null,
  evidence evidence_level not null,
  source_vendor_ids text[]
);


-- Institutional overrides layered above vendor data
create table ddi_overrides (
  id bigserial primary key,
  dataset_version text not null,                  -- becomes the synthesized dataset_version
  scope text not null check (scope in ('pair','class','pgx','modifier')),
  selector jsonb not null,                        -- match clauses; see Harmonizer
  action text not null check (action in ('replace','suppress','amend')),
  payload jsonb not null,                         -- new fields or fields to nullify
  rationale text not null,
  approver text not null,
  approved_at timestamptz not null default now()
);
```

create materialized view ddi_interaction_matrix as

     , evidence, confidence
from   ddi_interactions;


create index on ddi_interaction_matrix (dataset_version, drug1_code, drug2_code);
create index on ddi_interactions using gin (route_restriction);

## 3) Query & API Contracts (Refined 🚀)


We retain **gRPC + Protobuf** with **strict contracts**.&#x20;


```proto
syntax = "proto3";
package kb5.v1;


message PatientContext {
  map<string,string> pgx;       // {"CYP2D6":"ultrarapid"}
  string hepatic_stage;         // e.g., "ChildPugh_B"
  string renal_stage;           // e.g., "CKD_3b"
  string age_band;              // "adult", "older_adult"
}


message InteractionCheckRequest {
  string transaction_id = 1;
  repeated string drug_codes = 2;  // RxNorm/ATC normalized by KB-7
  PatientContext patient_context = 3;
  string dataset_version = 4;      // Evidence Envelope pin
  bool expand_classes = 5;         // resolve ATC classes to members
  bool include_contextuals = 6;    // include route/dose-specific flags
}


message InteractionDetail {
  string drug1_code = 1;
  string drug2_code = 2;
  string severity = 3;
  string mechanism = 4;
  string clinical_effects = 5;
  string management_strategy = 6;
  string evidence = 7;
  double confidence = 8;
  map<string,string> qualifiers = 9;   // route, dose thresholds, acute/chronic
  repeated string sources = 10;        // provenance
}


message ConflictTrail {
  string synthesized_from_version = 1; // vendor_version -> harmonized_version
  repeated string vendor_evidence_ids = 2;
  repeated string overrides_applied = 3; // institutional rule ids
}


message InteractionCheckResponse {
  string transaction_id = 1;
  string dataset_version = 2;
  repeated InteractionDetail interactions = 3;
  ConflictTrail conflict_trail = 4;
}
```


* **SLO**: p95 **< 80 ms** maintained (Redis + materialized view first; fallback to SQL path).&#x20;
* **Hard-stop predicate**: `severity == contraindicated` → immediate **FAIL** to CAE with management strategy.

## 4) Read Path (Critical-Hot)


1. **Normalize** (KB-7): codes → canonical RxCUI/ATC.
2. **Redis lookup**: `ddi:{dataset_version}:{min(drugA,drugB)}:{max(drugA,drugB)}` → direct hit returns InteractionDetail(s).
3. **SQL fallback**: **`ddi_interaction_matrix`** by (dataset\_version, drug1\_code, drug2\_code).&#x20;
4. **Contextual filter**: apply route/dose/PGx qualifiers in memory (fast).
5. **Assemble Evidence Envelope + ConflictTrail** (sources + overrides list).

## 6) Harmonizer & Versioning (Double-Buffered 📦)


* **Vendor ingest → Staging dataset\_version (e.g., `2025Q3.vendorA`)**
* **Harmonizer**:


  * Normalize to KB-7 codes (RxNorm/ATC)
  * **Merge** multi-vendor rows → choose **max severity**, most recent evidence, **compose management** (or mark conflict)
  * Apply **institutional overrides** (replace/suppress/amend) → **`2025Q3.harmonized`**
  * Emit **Provenance Map** per interaction (vendor IDs + override IDs)
* **Cutover**: Blue/Green switch of **`current_dataset_version`**, invalidate/refresh Redis hotset. (You proposed cache invalidation tied to ETL—retained.)&#x20;

## 7) Caching Strategy (Hot/Warm 🔥)


* **Hot**: Redis for top 50k pairs (Pareto of request distribution).
* **Warm**: Postgres `ddi_interaction_matrix`.
* **TTL**: None (pinned by version). Rotate on dataset cutover (atomic key-space swap).

## 13) Runbooks (Ops)

* **Rollback**: Reinstate previous `dataset_version` pointer. DB and Redis are version-qualified → instant revert.

## 14) Example SQL Helper (Fast Path)


```sql
create or replace function check_pair(v text, a text, b text)
returns table (severity ddi_severity, management_strategy text, evidence evidence_level, confidence numeric)
language sql stable as $$
  select severity, management_strategy, evidence, confidence
  from   ddi_interaction_matrix
  where  dataset_version = v
  and    drug1_code = least(a,b)
  and    drug2_code = greatest(a,b)
$$;
```

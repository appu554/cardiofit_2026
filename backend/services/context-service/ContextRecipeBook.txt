# Context Recipe Book - Data Gathering Edition

## Overview
This is the complete set of **Context Recipes** that live in the Context Service and define HOW to efficiently gather data required by Clinical Logic Recipes. These recipes orchestrate data fetching across multiple sources using our hybrid architecture.

## Enhanced Context Recipe Structure

```yaml
ContextRecipe:
  # Identity & Linkage
  id: unique-identifier-v1.0
  name: Human-readable name
  description: Data gathering purpose
  clinicalRecipeRefs: [List of clinical recipes this serves]
  
  # Activation
  triggers:
    - domain: "medication|order|lab"
    - action: "prescribe|modify|review"
    - urgency: "emergent|urgent|routine"
  
  # Data Requirements
  dataRequirements:
    realTime:  # Direct microservice calls
      - source: service-name
        endpoint: specific-endpoint
        data: fields-needed
        freshness: "<5min"
        timeout: 20ms
    
    federated:  # Apollo Federation GraphQL
      - source: federation
        query: GraphQL-query
        freshness: "<1hr"
        timeout: 50ms
    
    analytical:  # Elasticsearch
      - source: elasticsearch
        index: index-name
        query: ES-query
        freshness: "<24hr"
        timeout: 100ms
    
    external:  # Third-party integrations
      - source: external-system
        api: endpoint
        freshness: "<1hr"
        timeout: 200ms
  
  # Performance Optimization
  performanceStrategy:
    parallelization: true|false
    cachingStrategy:
      l1_ttl: seconds
      l2_ttl: seconds
    prefetchPatterns: [Predictive loading]
    fallbackStrategy: [Degradation plan]
  
  # Quality of Service
  qosTier: "platinum|gold|silver|bronze"
  maxLatency: milliseconds
  
  # Data Assembly
  assemblyStrategy:
    streaming: true|false
    partialResults: true|false
    mergeLogic: "How to combine sources"
```

### 1. MEDICATION CONTEXT RECIPES

#### Context Recipe 1.1: Basic Medication Safety Context
```yaml
id: medication-safety-base-context-v1.0
name: "Basic Medication Safety Data Gathering"
clinicalRecipeRefs: 
  - "medication-safety-base-v3.0"
  - "medication-safety-interactions-v3.0"
triggers:
  - domain: "medication"
  - action: ["prescribe", "modify", "renew"]

dataRequirements:
  federated:
    query: |
      query MedicationSafetyBase($patientId: ID!, $timestamp: DateTime!) {
        patient(id: $patientId) {
          demographics {
            age
            weight
            gender
            pregnancyStatus
            lactationStatus
          }
          allergies {
            substance
            type
            severity
            reactions
            verificationStatus
            lastUpdated
          }
        }
      }
    freshness: "<1hr"
    timeout: 40ms
    
  realTime:
    - source: "medication-service"
      endpoint: "/active-medications"
      data: |
        {
          patientId
          includePending: true
          fields: [drugCode, genericName, dose, frequency, 
                  startDate, prescriber, lastAdministered]
        }
      freshness: "<5min"
      timeout: 30ms
    
    - source: "order-service"
      endpoint: "/recent-orders"
      data: |
        {
          patientId
          orderTypes: ["MEDICATION"]
          status: ["PENDING", "ACTIVE"]
          window: "24hr"
        }
      freshness: "real-time"
      timeout: 20ms

  analytical:
    - source: "elasticsearch"
      index: "medication-history"
      query: |
        {
          "bool": {
            "must": [
              {"term": {"patientId": "$patientId"}},
              {"range": {"date": {"gte": "now-90d"}}}
            ]
          },
          "aggs": {
            "adherence": {
              "avg": {"field": "adherenceRate"}
            }
          }
        }
      freshness: "<24hr"
      timeout: 50ms

performanceStrategy:
  parallelization: true
  cachingStrategy:
    l1_ttl: 300  # 5 minutes
    l2_ttl: 3600  # 1 hour
  fallbackStrategy:
    - "Use cached allergies if <24hr old"
    - "Skip adherence data if unavailable"
    
qosTier: "gold"
maxLatency: 80
```

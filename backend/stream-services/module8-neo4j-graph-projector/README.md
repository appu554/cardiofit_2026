# Neo4j Graph Projector Service

**Module 8 Storage Projector**: Projects graph mutations from Kafka to Neo4j patient journey graphs.

## Overview

The Neo4j Graph Projector consumes `GraphMutation` objects from the `prod.ehr.graph.mutations` Kafka topic and executes Cypher queries to build rich patient journey graphs in Neo4j.

Unlike other Module 8 projectors that consume enriched events, this projector works with **pre-defined graph mutations** generated by Module 6 semantic analysis services.

## Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                    prod.ehr.graph.mutations                     │
│              (GraphMutation objects from Module 6)              │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│             Neo4j Graph Projector (This Service)                │
│                                                                 │
│  ┌──────────────────────┐      ┌──────────────────────────┐    │
│  │  Kafka Consumer      │ ───→ │  Cypher Query Builder    │    │
│  │  (50 msg batches)    │      │  (MERGE/CREATE/REL)      │    │
│  └──────────────────────┘      └──────────────────────────┘    │
│                                            ↓                    │
│                              ┌──────────────────────────┐       │
│                              │  Neo4j Transaction       │       │
│                              │  Execute Cypher Queries  │       │
│                              └──────────────────────────┘       │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                         Neo4j Database                          │
│              Patient Journey Graph (cardiofit DB)               │
│                                                                 │
│  Nodes: Patient, ClinicalEvent, Condition, Medication,         │
│         Procedure, Department, Device                           │
│                                                                 │
│  Relationships: HAS_EVENT, HAS_CONDITION, PRESCRIBED,          │
│                 UNDERWENT, NEXT_EVENT, TRIGGERED_BY,            │
│                 LOCATED_IN, MEASURED_BY                         │
└─────────────────────────────────────────────────────────────────┘
```

## Key Differences from Other Projectors

| Aspect | Neo4j Graph Projector | Other Projectors |
|--------|----------------------|------------------|
| **Input Topic** | `prod.ehr.graph.mutations` | `prod.ehr.events.enriched` |
| **Input Model** | `GraphMutation` | `EnrichedClinicalEvent` |
| **Processing** | Execute pre-defined Cypher queries | Parse and transform events |
| **Semantic Analysis** | PRE-COMPUTED by Module 6 | Not required |
| **Operations** | MERGE nodes + CREATE relationships | INSERT/UPSERT records |
| **Target** | Graph database (Neo4j) | Relational/document stores |

## Graph Schema

### Node Types

1. **Patient**: Patient demographic and clinical summary
   - Properties: `nodeId`, `firstName`, `lastName`, `dateOfBirth`, `lastUpdated`

2. **ClinicalEvent**: Individual clinical observations or events
   - Properties: `nodeId`, `patientId`, `eventType`, `timestamp`, `lastUpdated`

3. **Condition**: Diagnoses and clinical conditions
   - Properties: `nodeId`, `patientId`, `conditionCode`, `conditionName`, `onsetDate`

4. **Medication**: Prescribed medications
   - Properties: `nodeId`, `patientId`, `medicationCode`, `medicationName`, `startDate`

5. **Procedure**: Medical procedures performed
   - Properties: `nodeId`, `patientId`, `procedureCode`, `procedureName`, `performedDate`

6. **Department**: Hospital departments
   - Properties: `nodeId`, `departmentName`, `departmentType`

7. **Device**: Medical devices
   - Properties: `nodeId`, `deviceId`, `deviceType`, `manufacturer`

### Relationship Types

1. **HAS_EVENT**: `(Patient)-[:HAS_EVENT]->(ClinicalEvent)`
   - Patient's clinical events

2. **HAS_CONDITION**: `(Patient)-[:HAS_CONDITION]->(Condition)`
   - Patient's diagnoses

3. **PRESCRIBED**: `(Patient)-[:PRESCRIBED]->(Medication)`
   - Patient's medications

4. **UNDERWENT**: `(Patient)-[:UNDERWENT]->(Procedure)`
   - Patient's procedures

5. **NEXT_EVENT**: `(ClinicalEvent)-[:NEXT_EVENT]->(ClinicalEvent)`
   - Temporal sequence of events

6. **TRIGGERED_BY**: `(ClinicalEvent)-[:TRIGGERED_BY]->(Condition)`
   - Event causality

7. **LOCATED_IN**: `(Patient)-[:LOCATED_IN]->(Department)`
   - Patient location

8. **MEASURED_BY**: `(ClinicalEvent)-[:MEASURED_BY]->(Device)`
   - Device measurements

### Neo4j Browser Queries

Access Neo4j Browser at `http://localhost:7474` and run:

#### 1. Patient Journey (Chronological Events)

```cypher
MATCH (p:Patient {nodeId: 'P12345'})-[:HAS_EVENT]->(e:ClinicalEvent)
RETURN p, e
ORDER BY e.timestamp;
```

#### 2. Temporal Event Sequence

```cypher
MATCH path = (e1:ClinicalEvent)-[:NEXT_EVENT*]->(e2:ClinicalEvent)
WHERE e1.patientId = 'P12345'
RETURN path
LIMIT 100;
```

#### 3. Clinical Pathway Analysis

```cypher
MATCH (p:Patient)-[:HAS_CONDITION]->(c:Condition),
      (p)-[:HAS_EVENT]->(e:ClinicalEvent)-[:TRIGGERED_BY]->(c)
WHERE p.nodeId = 'P12345'
RETURN p, c, e
ORDER BY e.timestamp;
```

#### 4. Patient Medications

```cypher
MATCH (p:Patient {nodeId: 'P12345'})-[:PRESCRIBED]->(m:Medication)
RETURN p, m
ORDER BY m.startDate DESC;
```

#### 5. Department Activity

```cypher
MATCH (d:Department {nodeId: 'DEPT_ICU'})<-[:LOCATED_IN]-(p:Patient)-[:HAS_EVENT]->(e:ClinicalEvent)
RETURN d, p, e
ORDER BY e.timestamp DESC
LIMIT 50;
```

#### 6. Device Measurements

```cypher
MATCH (dev:Device {nodeId: 'DEV_MONITOR_001'})<-[:MEASURED_BY]-(e:ClinicalEvent)
RETURN dev, e
ORDER BY e.timestamp DESC
LIMIT 100;
```

#### 7. Patient Summary

```cypher
MATCH (p:Patient {nodeId: 'P12345'})
OPTIONAL MATCH (p)-[:HAS_CONDITION]->(c:Condition)
OPTIONAL MATCH (p)-[:PRESCRIBED]->(m:Medication)
OPTIONAL MATCH (p)-[:HAS_EVENT]->(e:ClinicalEvent)
RETURN p,
       collect(DISTINCT c) as conditions,
       collect(DISTINCT m) as medications,
       count(DISTINCT e) as event_count;
```

## Performance

### Target Metrics

- **Throughput**: ~500 mutations/sec
- **Batch Size**: 50 mutations
- **Batch Timeout**: 5 seconds
- **Query Latency**: <100ms for patient journey queries

### Optimization

1. **Constraints**: Ensure unique constraints on `nodeId` for fast lookups
2. **Indexes**: Create indexes on frequently queried properties (`timestamp`, `patientId`)
3. **Batch Processing**: Process mutations in batches with Neo4j transactions
4. **Connection Pool**: Configure max_connection_pool_size=50

## Integration

This service is part of Module 8 (Multi-Sink Storage Layer):

1. **Module 6**: Generates `GraphMutation` objects via semantic analysis
2. **Kafka Topic**: `prod.ehr.graph.mutations` (16 partitions, 30 days retention)
3. **This Service**: Executes graph mutations in Neo4j
4. **Downstream**: Patient journey visualizations, clinical pathway analysis

## Use Cases

1. **Patient Journey Visualization**: Chronological view of patient's clinical events
2. **Clinical Pathway Analysis**: Identify care patterns and pathways
3. **Temporal Analysis**: Track event sequences and causality
4. **Population Health**: Aggregate patterns across patient cohorts
5. **Clinical Research**: Graph-based analytics on patient data

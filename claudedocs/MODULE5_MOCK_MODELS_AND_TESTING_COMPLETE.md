### Why Predictions Are ~94% for Everyone

**Observation**: Healthy patient (age 20, perfect vitals) shows 94% sepsis risk, same as critical patient (age 42, septic shock).

**Explanation**:
- Models learned patterns from completely **random synthetic data**
- No real clinical knowledge embedded
- Training labels were randomly generated (not actual sepsis outcomes)
- XGBoost learned whatever random correlations existed in fake data
- Result: All predictions cluster around baseline prevalence (~94%)

### Legitimate Purpose of Mock Models

These models serve a **software engineering purpose**:
1. ✅ Infrastructure Testing: Prove ONNX Runtime integration works
2. ✅ Performance Benchmarking: Measure throughput (95K+ pred/sec)
3. ✅ API Validation: Verify model loading, feature extraction, prediction pipeline
4. ✅ Deployment Testing: Test model packaging, deployment scripts

They do **NOT** prove:
- ❌ Clinical predictions are accurate
- ❌ Models can identify real sepsis cases
- ❌ Risk scores have clinical validity
- ❌ Models are ready for production use

## 📊 Model Specifications

### 4 Clinical Risk Models

| Model | File | Input | Output | Purpose |
|-------|------|-------|--------|---------|
| Sepsis Risk | `sepsis_risk_v1.0.0.onnx` | 70 features | 2 tensors | Sepsis onset prediction (48hr window) |
| Deterioration | `deterioration_risk_v1.0.0.onnx` | 70 features | 2 tensors | Clinical deterioration (6-24hr window) |
| Mortality | `mortality_risk_v1.0.0.onnx` | 70 features | 2 tensors | In-hospital mortality prediction |
| Readmission | `readmission_risk_v1.0.0.onnx` | 70 features | 2 tensors | 30-day unplanned readmission |

### Input Features (70-Dimensional Vector)

**Demographics**: Age, gender, weight, height, BMI

**Vital Signs**: Heart rate, respiratory rate, temperature, blood pressure (systolic/diastolic), oxygen saturation, MAP

**Lab Values**: WBC, hemoglobin, platelets, creatinine, BUN, glucose, sodium, potassium, lactate, bilirubin

**Clinical Scores**: SOFA, qSOFA, NEWS2, shock index, age-adjusted Charlson

**Medications**: Vasopressor use, sedation, antibiotics, insulin

**Ventilation**: Mechanical ventilation status, FiO2, PEEP

**Other**: ICU admission, time since admission, prior ICU stays

### Output Format

ONNX models return **2 tensors**:
1. **Tensor[0]**: Class labels (INT64) - predicted class (0 or 1)
2. **Tensor[1]**: Probabilities (FLOAT) - [negative_prob, positive_prob]

**We use Tensor[1]** for risk scores (positive class probability).

### Proposed Real Model Training Pipeline

#### Phase 1: Data Extraction (Week 1)
1. Set up GCP BigQuery connection
2. Extract cohort data:
   - Sepsis cases: ICD-10 codes + clinical criteria (qSOFA ≥2, lactate >2)
   - Deterioration: ICU transfers, code blue events
   - Mortality: In-hospital deaths
   - Readmission: 30-day unplanned returns

**Minimum Sample Sizes**:
- Sepsis: 10,000 positive cases
- Deterioration: 8,000 positive cases
- Mortality: 5,000 positive cases
- Readmission: 15,000 positive cases

#### Phase 2: Feature Engineering (Week 2)
1. Extract same 70 features from MIMIC-IV tables:
   - `chartevents` → Vital signs
   - `labevents` → Lab values
   - `prescriptions` → Medications
   - `diagnoses_icd` → Comorbidities
   - `icustays` → ICU admission data

2. Implement time windows:
   - First 6 hours of ICU admission
   - 12-hour rolling windows
   - 24-hour aggregations

3. Handle missing data:
   - Forward fill vitals (last known value)
   - Mean imputation for labs
   - Missingness indicators as features

#### Phase 3: Model Training (Week 3)
1. Split data: 70% train, 15% validation, 15% test
2. Train XGBoost models:
   - Hyperparameter tuning (GridSearch)
   - Cross-validation (5-fold)
   - Class balancing (SMOTE or class weights)

3. Validation metrics:
   - AUROC ≥ 0.85 (required)
   - Sensitivity ≥ 0.80 (catch 80% of cases)
   - Specificity ≥ 0.75 (avoid too many false alarms)
   - Calibration: Brier score, calibration plots

#### Phase 4: Export to ONNX (Week 4)
1. Export trained XGBoost models to ONNX format
2. Validate ONNX Runtime inference matches XGBoost
3. Replace mock models in `models/` directory
4. Re-run all test suites

5. Update model metadata:

   ```python
   meta_auroc.key = 'test_auroc'

   ```

#### Phase 5: Clinical Validation (Week 5)
1. External validation on hold-out MIMIC-IV test set
2. Temporal validation (train on 2008-2016, test on 2017-2019)
3. Subgroup analysis:
   - Age groups (18-40, 41-65, 66+)
   - Gender
   - Sepsis source (pulmonary, abdominal, urinary)

4. Calibration assessment:
   - Calibration plots (predicted vs observed risk)
   - Hosmer-Lemeshow test
   - Recalibration if needed
